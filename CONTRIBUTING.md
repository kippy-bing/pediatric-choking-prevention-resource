# Contributing to Pediatric Choking Prevention Resource

Thank you for your interest in contributing to this open-source educational resource. Your contributions will help save children's lives by improving access to accurate choking prevention and response information.

## Who Can Contribute?

- **Healthcare professionals** (doctors, nurses, EMTs, first responders)
- **Parents and caregivers** with lived experience
- **Educators and childcare providers**
- **Translators and localization experts**
- **Web developers and designers**
- **Researchers and academics**
- **Anyone passionate about child safety**

## Types of Contributions

### 1. Educational Content
- Add new choking hazards based on recent research or reports
- Improve existing content with updated medical guidelines
- Create age-specific prevention strategies
- Provide visual aids (diagrams, illustrations)
- Share personal stories (with appropriate privacy considerations)

### 2. Technical Improvements
- Enhance the interactive demonstration tool
- Improve website accessibility
- Add multilingual support
- Optimize repository structure
- Fix bugs or improve performance

### 3. Community Engagement
- Answer questions in discussions
- Review pull requests
- Help triage issues
- Share the resource with your network

### 4. Documentation
- Improve clarity of existing documentation
- Add translations of documentation
- Create tutorials or how-to guides

## Getting Started

### Prerequisites
- A GitHub account
- Basic understanding of Markdown for content contributions
- For code contributions: familiarity with HTML, CSS, JavaScript, and Git

### First-Time Contributors
1. **Fork the repository** to your GitHub account
2. **Clone your fork** locally
3. **Create a new branch** for your contribution
4. **Make your changes**
5. **Test your changes** (if applicable)
6. **Commit and push** to your fork
7. **Open a pull request** to the main repository

## Contribution Guidelines

### Educational Content Standards
- All medical information must be evidence-based and cite reputable sources
- Content should be clear, concise, and accessible to non-medical audiences
- Use plain language and avoid unnecessary medical jargon
- Include age-specific recommendations where appropriate
- Follow established first aid guidelines (American Heart Association, Red Cross, etc.)

### Code Contribution Standards
- Follow existing code style and patterns
- Ensure responsive design for mobile and desktop
- Maintain accessibility standards (WCAG 2.1 AA)
- Include appropriate comments
- Update documentation as needed

### Issue Reporting
When reporting a potential choking hazard, safety concern, or bug, please include:
1. **Clear description** of the issue
2. **Age group** affected (infant, toddler, child, adolescent)
3. **Source/reference** if available
4. **Severity assessment** (high, medium, low risk)
5. **Suggested solution** (optional)

### Pull Request Process
1. **Ensure your PR addresses a single issue** or adds a single feature
2. **Update documentation** if you change functionality
3. **Add tests** for code changes (if applicable)
4. **Request review** from maintainers
5. **Respond to feedback** promptly

## Review Process

### Content Review
All educational content undergoes medical review:
1. **Initial review** by maintainers for completeness and clarity
2. **Medical review** by healthcare professionals for accuracy
3. **Final approval** before merging

### Code Review
All code changes undergo technical review:
1. **Automated checks** for syntax and style
2. **Manual review** for functionality and security
3. **Accessibility review** for compliance

## Recognition

Contributors are recognized in several ways:
- **GitHub contributors list**
- **Acknowledgments in release notes**
- **Featured contributor spotlights** (with permission)
- **Digital badges** for significant contributions
